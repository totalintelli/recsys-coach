# 계층 불릿(●/○ 중첩 목록) 렌더링 복구

## Context

사용자는 답변이 첨부 이미지처럼 **계층 불릿**(상위 ● disc, 하위 ○ circle, 들여쓰기로 계층 표현)으로 렌더링되길 원한다. CSS·프롬프트·markdown-it-py는 이미 계층 목록을 지원하도록 갖춰져 있는데, **실제 화면에서는 하위 불릿이 사라지고 모든 항목이 같은 레벨(●)로 평탄화**된다.

### 근본 원인 (정적 분석으로 확정)

`src/qa_chain.py`의 `_clean_answer()` (line 72-79) 마지막 줄:

```python
text = re.sub(r" {2,}", " ", text)   # line 78
```

이 정규식은 **2칸 이상의 모든 연속 공백을 1칸으로 압축**한다. 줄 끝뿐 아니라 **줄머리 들여쓰기까지** 무차별 압축한다.

- 시스템 프롬프트(line 32)는 LLM에게 하위 항목을 `  - `(2칸 들여쓰기)로 출력하라고 지시한다.
- 그런데 `_clean_answer`가 그 2칸을 1칸으로 죽인다.
- CommonMark(markdown-it-py)는 **1칸 들여쓰기를 중첩으로 인정하지 않아** 같은 레벨 목록으로 처리한다.
- 결과: `<ul><li>...<ul><li>` 중첩이 생기지 않고 모두 최상위 `<li>` → ○ 하위 계층이 사라진다.

즉 프롬프트와 후처리가 서로 충돌하는 자기모순이다. CSS(line 216-221)와 중첩 규칙은 정상이므로, **들여쓰기를 보존하면 이미지와 동일하게 렌더링된다.**

추가 영향: 같은 줄의 `re.sub(r"\n{3,}", "\n\n", text)`는 목록과 무관하니 유지. 단 공백 압축이 코드 블록 내부(들여쓰기 보존 필요) 정합성에도 영향을 줄 수 있으나, 본 변경은 들여쓰기를 더 보존하는 방향이라 안전하다.

## 변경 사항

### 1. `src/qa_chain.py` — `_clean_answer()` 공백 압축이 줄머리 들여쓰기를 보존하도록 수정

목표: **줄 중간/끝의 중복 공백만 압축하고, 줄머리 들여쓰기 공백은 보존**한다.

`_clean_answer` (line 72-79) 내 line 78의

```python
text = re.sub(r" {2,}", " ", text)
```

을 줄머리 들여쓰기를 건드리지 않는 형태로 교체한다. 권장 방식 — 줄 시작(`^`, MULTILINE)이 아닌 위치의 중복 공백만 압축:

```python
text = re.sub(r"(?<=\S) {2,}", " ", text)  # 비공백 뒤의 중복 공백만 압축 (줄머리 들여쓰기 보존)
```

`(?<=\S)` lookbehind로 "직전 문자가 비공백일 때만" 압축하므로, 줄머리(직전이 `\n` 또는 문자열 시작)의 들여쓰기 공백은 그대로 남는다. 본문 중간의 우발적 더블스페이스는 여전히 정리된다.

> 참고: `_clean_text()` (line 61-69)에도 동일한 `re.sub(r" {2,}", " ", text)` (line 68)가 있으나, 이는 **검색 context용 청크 전처리**라 화면 목록 렌더링과 무관하다. 일관성을 위해 같이 바꿀 수 있으나 **필수는 아니다** — 본 작업 범위에서는 `_clean_answer`만 수정한다(최소 변경).

### 2. (선택) 헤더 색상 — 현재 황금색 유지

이미지는 보라색 헤더지만, 사용자의 요청(B)은 "계층 불릿이 안 나온다"이므로 **색상은 범위 밖**. 기존 `#f0b429`(line 133/142/149/157)는 그대로 둔다. 사용자가 색까지 원하면 별도 처리.

## 검증

환경의 Bash classifier가 일시적으로 불가했으므로, 구현 시 아래로 반드시 재확인한다.

1. **단위 재현 스크립트** (수정 전/후 비교):
   ```bash
   source .venv/bin/activate
   python3 -c '
   from src.qa_chain import _md_to_html, _clean_answer
   md = "## Train-Test Split\n- Train data\n  - 4개월 데이터\n- Test data\n  - 일주일 데이터\n  - Cold-start 제외"
   cleaned = _clean_answer(md)
   print("repr:", repr(cleaned))           # 하위 항목 줄머리에 "  "(2칸)이 남아야 함
   print(_md_to_html(cleaned).split("<style>")[0])
   '
   ```
   - **기대:** `repr`에서 `\n  - 4개월` 처럼 2칸 들여쓰기 보존. HTML에 중첩 `<ul>...<li>...<ul><li>` 구조가 생성됨.
   - 수정 전에는 `\n - 4개월`(1칸)으로 평탄화되어 중첩 `<ul>`이 없음 → 대비로 회귀 여부 확인.

2. **본문 중복 공백 압축 회귀 확인:** `_clean_answer("a  b   c")` → `"a b c"` 로 여전히 정리되는지(줄 중간 압축은 유지) 확인.
