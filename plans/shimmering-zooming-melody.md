# 프롬프트 예시에서 '파일명' 문자열 제거

## Context

`[파일명 N page]` 형식 지시문을 LLM이 템플릿 변수가 아닌 출력 문자열로 오해해 실제 답변에 "파일명"이 그대로 나타난다. context 태그는 이미 실제 값(`[규칙서.pdf 3 page]`)으로 전달되므로, 프롬프트 예시를 구체적인 가상 파일명으로 바꾸면 LLM이 형식을 올바르게 모방한다.

## 수정 대상

**`src/qa_chain.py` 라인 49** — 시스템 프롬프트 지시문 한 줄

```python
# 현재
"각 문장 끝에 해당 내용의 출처를 공백 한 칸을 두고 [파일명 N page] 형식으로 반드시 붙이세요. "

# 변경 후
"각 문장 끝에 해당 내용의 출처를 공백 한 칸을 두고 [document.pdf 3 page] 같은 형식으로 반드시 붙이세요. "
```

## 검증

`streamlit run app.py` 실행 후 PDF 2개 업로드 → 질문 → 답변에 "파일명"이라는 단어가 나타나지 않고 실제 파일명이 출력되는지 확인.
